\chapter{Introducción}

\section{Propósito del documento}
El primer entregable del proyecto consiste en la elaboración del documento de requisitos del producto ALMA[cite: 4]. Este documento constituye la base técnica del trabajo posterior, redactado con rigor, claridad y trazabilidad[cite: 5]. Su propósito es definir explícitamente qué debe hacer el producto, en qué condiciones debe hacerlo, qué restricciones debe cumplir y cómo se verificará cada requisito[cite: 6]. 

\section{Alcance del producto}
El producto ALMA se concibe como un dispositivo con capacidades de interacción por voz, reproducción de audio, control de otros dispositivos e interfaz gráfica táctil[cite: 86, 88]. En esta etapa, el documento describe únicamente los requisitos técnicos y el desempeño esperado del producto, y no incluye decisiones de implementación detallada, marcas comerciales, ni actividades de gestión del proyecto[cite: 8, 17, 68, 103].

\section{Metodología de construcción de requisitos}
Para elaborar esta lista de requisitos, el equipo analizó en primer lugar la descripción del producto ALMA para extraer las necesidades implícitas y explícitas[cite: 73]. Posteriormente, estas necesidades se organizaron en las categorías exigidas (funcionales, no funcionales, ambientales, GUI y estándares)[cite: 74]. A continuación, se revisaron normativas y productos similares para identificar exigencias razonables[cite: 75]. Finalmente, la versión inicial se depuró para eliminar ambigüedades, consolidar redundancias y asignar criterios de aceptación y métodos de verificación concretos a cada requisito[cite: 77, 79].
